# LEO TDOA Simulation

LEO 卫星 TDOA 定位仿真程序，用于模拟低轨卫星对地面发射源的接收、TDOA 定位、链路预算、GDOP 几何精度和蒙特卡洛统计分析。

## 安装

```bash
pip install -r requirements.txt
```

说明：

- `tkinter` 为 Python 标准库自带 GUI 组件，通常无需单独安装。
- 如需打包为 exe，需额外安装 `pyinstaller`。

## 启动

```bash
python leo_tdoa_simulation.py
```

程序启动后默认窗口最大化。

## 当前默认参数

- 频率: `14.25 GHz`
- 带宽: `62.5 MHz`
- 发射 EIRP: `62 dBm`
- 相控阵规模: `34 x 44`
- 卫星高度: `600 km`
- 卫星数量: `3`
- 卫星间距: `100 km`
- 定位精度要求: `1000 m`
- 蒙特卡洛次数: `20`

## 界面参数说明

### 信号参数

- `频率 (GHz)`: 发射信号载频。
- `带宽 (MHz)`: 用于 TDOA 时间分辨率估计，带宽越大，理论时间精度越高。
- `发射EIRP (dBm)`: 发射源等效全向辐射功率。
- `天线阵元 Nx / Ny`: 发射端相控阵的阵元规模，会影响方向图和离轴增益损失。

### 卫星参数

- `卫星高度 (km)`: 卫星轨道高度。
- `卫星数量`: 参与 TDOA 的卫星数。
- `卫星间距 (km)`: 当前程序里表示中心参考星到外环卫星的距离，不是外环相邻卫星之间的弧长。
- `接收天线增益 (dBi)`: 卫星接收端天线增益。
- `噪声系数 (dB)`: 接收链路噪声系数。

### 发射源位置

- `X偏移 / Y偏移 (km)`: 地面发射源相对场景原点的平面偏移。

### 定位与统计

- `定位精度要求 (m)`: 目标定位误差门限。
- `蒙特卡洛次数`: 每个卫星间距采样点重复仿真的次数，仅用于蒙特卡洛分析。

## 当前模型说明

- 地面发射源定位按 `二维平面 (x, y)` 解算，固定 `z = 0`。
- 卫星布局采用 `1` 颗中心参考星加外环卫星的方式。
- 当卫星数量为 `3` 时，外环两颗卫星方位角固定为 `0°` 和 `90°`。
- 当卫星数量大于等于 `4` 时，外环 `N-1` 颗卫星按 `360° / (N-1)` 均匀分布。
- GDOP 基于地面二维 TDOA 几何矩阵计算，用来衡量几何布局优劣。
- 最终定位误差不仅取决于 GDOP，也会受到 SNR、天线方向图深陷和随机测量噪声影响。

## 图表说明

### 单次仿真页面

- `XZ 平面投影`: 展示卫星、真实位置和估计位置的侧视关系。
- `XY 平面投影`: 展示地面平面内的真实位置、估计位置和精度要求圆。
- `定位精度分析`: 对比当前定位误差和目标精度要求。
- `各卫星接收信噪比`: 查看每颗卫星的 SNR 水平。

### 蒙特卡洛分析页面

- `定位误差 vs 卫星间距`: 使用误差中位数作为主曲线，并显示 `25%-75%` 分位区间。
- `几何精度因子 vs 卫星间距`: 展示 GDOP 随间距变化趋势。
- `蒙特卡洛定位误差分布`: 汇总所有采样结果的误差分布。
- `链路最差点 vs 卫星间距`: 同时显示最差卫星 SNR 和最大天线损失，用来判断异常误差尖峰是否由链路坏点引起。

## 结果区说明

- 单次仿真结果会给出定位误差、GDOP、每颗卫星的接收功率、SNR、离轴角和 TDOA 测量值。
- 蒙特卡洛结果表使用 `中位误差 / P25 / P75 / GDOP` 统计口径，较均值更稳健。
- 结果区已关闭自动换行，并支持横向滚动，便于查看完整表格。

## 主要功能

- 地面发射源 TDOA 定位仿真
- 链路预算与 SNR 分析
- 二维地面 GDOP 计算
- 可配置次数的蒙特卡洛分析
- 基于中位数与分位区间的稳健误差统计
- 最差卫星 SNR 与最大天线损失可视化
- 交互式 GUI 参数面板和文本结果区

## 常见现象与排查建议

- `3` 颗卫星误差通常高于 `4+` 颗：
  当前模型虽然已经改为地面二维定位，但 `3` 颗卫星本质上仍然只有最小可解冗余，抗噪声和抗链路坏点能力明显弱于 `4+` 颗。
- 局部间距可能出现误差尖峰：
  这通常不是 GDOP 单独造成的，而是某些卫星在对应角度落入天线方向图深陷区，导致 SNR 降低、TDOA 误差放大。可结合“最差卫星 SNR”和“最大天线损失”图一起看。
- 蒙特卡洛曲线与单次仿真不完全一致：
  单次仿真是一次随机采样，蒙特卡洛曲线是多次采样后的统计结果，后者更适合判断趋势。
- 蒙特卡洛运行较慢：
  可先减小“蒙特卡洛次数”快速看趋势，再提高次数做最终统计。
- 结果区表格显示不全：
  结果区已支持横向滚动，拖动底部滚动条即可查看完整表格。

## 已知限制

- 当前定位模型只支持地面二维定位，不适用于真正的三维空中目标定位。
- `find_minimum_spacing()` 使用的是基于 GDOP 和时间精度的近似估算，不等价于完整蒙特卡洛最优解。
- 天线方向图会引入明显的非平滑链路坏点，因此某些间距下可能出现局部误差升高，这属于模型行为的一部分。
